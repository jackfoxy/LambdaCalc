Unit Tests

Did you know there is a Î»-abstraction you can use as the assert statement in unit tests? It is the boolean function on two variables for equivalence, sometimes known as iff or xnor.

We have implemented this function abstraction as xnor, and possibly not in the most efficient manner. Try your hand at developing more efficient implementations. You have unit tests to help you.

There are a number of example unit tests here.

The tests make use of a very minimalistic language defined in the untyped lambda calculus (list is in dependency order):

  • prelude includes the bottom and fix abstraction values required by the untyped recursive lambda calculus.
  • boolean includes the boolean abstractions and values for the most common boolean operations.
  • tuple the pair tuple and operations.
  • numbers some natural number abstractions and operations on natural numbers.
  • lists the singly linked list abstraction.

Most of the language elements have been described by Pierce and elsewhere, with the list terms being a unique implementation (that is not to say no one else has done it before) implementing nil as bottom in order to function with fix in the untyped recursive lambda calculus.
